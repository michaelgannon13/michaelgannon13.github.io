The Differences between Angular 1 and Angular 2

Identifying the main differences between Google's Angular 1 and 2

Posted by Michael Gannon on March 13, 2017

TL;DR

Angular 2...
  1. Performs better than Angular 1.
  2. Is easier to use.
  3. Is future proofed with ES6.
  4. Uses emerging web standards.
  5. Uses lots of new syntax.
  6. Does not facilitate two way data binding.
  7. Uses the Javascript precompiler, Typescript, as it's default language.
  8. It doesn't have controllers, it has component. These are made up of a views and Javascript classes.
  9. It has a new compnent router.

Context and Angular 1

Google in their infinite wisdom did a complete re-write of Angular to make it easier to use, more future proof and more efficient. However, with this new release there has also been a lot of changes to the syntax, the framework architecture and the way we interact with it, so this means it's quite different from Angular 1 (but not dramaticallt different so be not afraid!).

Angular 1 is built using an M.V.C architechture, which stands for Model, View and Controller. This is how Angular code is kept modular. The Model contains our raw data whether it be in JSON format or XML format. Our View contains all of our HTML templates which will be rendered on the screen and will be populated with Model data. Our Controllers retrieve our raw data (from our Model) and contains the logic that decides what is to be done with this data. Angular is very clever.

Above we can see the view on the left displaying a HTML table, which is receiving the table data from our controller, which is retrieving this data from a Javascript object.....Anyway none of this matters because now Angular 2 is out. No a lot of it is still relevent

Angular 2

Angular is made up of Components, Directives, Services and Routers.
The components encapsulates the template, the data and the behaviour of the view. A component can contain components and can also be reused e.g. the same navbar can be used repeatedly. Basically a component is a Javascript class that has properties and methods. Below is an visual of what a component is in relation to the DOM.

Step 1

Enable USB Debugging on device

It will usually be in Settings > About Phone > Build Number > Tap it 7 times to become a Developer. Then you will see Developer options is available in the Settings menu so click it and ensure USB debugging is set to on. Ensure Google Chrome is open on your mobile and your desktop simultaneously.

Step 2

Connect your device to your desktop

Connect your mobile device to the desktop, open chrome and open developer tools (Cmd + alt + i / Ctrl + shift + i). Click the 3 dots on the top right courner > More tools > Remote devices.

Step 3

Select device and open a tab

In the remote devices section, of developer tools, on the left hand side you’ll see a list of remote devices connected to the computer and a settings tab. Click Settings and ensure Discover USB devices is ticked.

Next select your device from the menu on the left, in this case it’s SM-G920F. Enter the url of the website you wish to debug on your mobile, hit open and the site will be launched on your mobile. Click 'Inspect' to the right of the site you just opened and it will allow you to traverse the DOM of your mobile phone's browser from your desktop's browser.
